🎮 插件簡介

DungeonSystem 是一個功能完整的 Minecraft 副本系統插件，為伺服器提供豐富的 PvE 內容體驗。

🏛️ 副本系統

  • 普通副本：擊殺指定目標怪物完成
  • 波次副本：分波次清理怪物
  • 支援等級要求和人數限制

👥 組隊系統

  • 自定義組隊功能
  • MMOCore 整合支援
  • 隊伍聊天和管理

🎫 入場券系統

  • 右鍵點擊入場券進入副本
  • 只有隊長能使用入場券
  • 自動檢查隊伍條件

💖 復活系統

  • 普通復活裝置（需等待10秒）
  • 高級復活裝置（立即復活）
  • 隊友復活機制

📦 安裝要求

1

核心要求

Minecraft 版本：1.16+

MythicMobs DecentHolograms PlaceholderAPI
2

可選依賴

MMOCore

整合組隊系統（可選）

3

安裝步驟

  1. 將插件 JAR 文件放入 plugins 資料夾
  2. 重啟伺服器
  3. 編輯生成的配置文件
  4. 使用 /dungeon reload 重載配置

🎯 詳細功能

副本類型

普通副本

  • 擊殺指定目標怪物完成
  • 支援多種怪物配置
  • 可設置等級要求和人數限制
  • 自動怪物縮放系統

波次副本

  • 分波次清理怪物
  • 支援強制下一波倒數
  • 每波可配置不同怪物
  • 音效與視覺反饋

組隊系統

/party create - 創建隊伍 /party invite <玩家> - 邀請玩家 /party accept - 接受邀請 /party leave - 離開隊伍 /party info - 查看隊伍信息

排行榜系統

  • 📊臨時排行榜：在玩家上方顯示30秒
  • 🏆永久排行榜：DecentHolograms 全息圖顯示
  • 📈詳細統計：完成次數、最佳時間、擊殺數
  • 🔄自動更新：實時更新排名數據

📝 指令大全

玩家指令

指令 權限 描述
/dungeon - 顯示幫助信息
/dungeon join <副本名稱> dungeon.join 加入副本
/dungeon list - 列出可用副本
/dungeon leave - 離開副本
/dungeon rank <副本名稱> - 顯示排行榜

管理員指令

指令 權限 描述
/dungeon reload dungeonsystem.admin 重載配置
/dungeon key <副本ID> [數量] dungeonsystem.admin 生成入場券
/dungeon revive <類型> [數量] dungeonsystem.admin 生成復活道具
/dungeon mob <參數> dungeonsystem.admin 添加怪物到副本

🔑 權限設置

基礎權限

permissions: dungeonsystem.admin: description: 管理員權限 default: op dungeon.join: description: 進入副本權限 default: false dungeonsystem.key.副本ID: description: 獲取特定副本入場券權限 default: false

權限組配置範例

groups: player: permissions: - dungeon.join - dungeonsystem.key.dungeon1 vip: permissions: - dungeon.join - dungeonsystem.key.* admin: permissions: - dungeonsystem.admin

⚙️ 配置示例

基本設置 (config.yml)

settings: exit-point: "world,0,64,0,0,0" # 副本退出點 revival-system-enabled: true # 復活系統開關 key-settings: cooldown-seconds: 2 # 鑰匙使用冷卻時間 party: system-type: "custom" # 組隊系統類型 custom/mmocore

普通副本配置

dungeons: dungeon_normal_1: type: "normal" level-required: 0 max-players: 4 display-name: "§e簡單副本" spawn-point: "world,100,64,100,0,0" death-waiting-area: "world,110,64,110,0,0" target-mob: "boss_goblin_king" mobs: - id: "dungeon_goblin" location: "world,120,64,120,0,0" amount: 3 radius: 5.0 level: 1 type: NORMAL

波次副本配置

dungeon_wave_1: type: "wave" level-required: 10 max-players: 5 display-name: "§b挑戰之塔" spawn-point: "world,200,64,200,0,0" death-waiting-area: "world,210,64,210,0,0" waves: total: 3 wave-1: - id: "dungeon_skeleton" location: "world,220,64,220,0,0" amount: 5 radius: 3.0 level: 2 type: NORMAL

❗ 常見問題

Q: 玩家無法進入副本？

A: 檢查以下項目：

  • 玩家是否在隊伍中且為隊長
  • 玩家等級是否達到要求
  • 副本是否被其他隊伍占用
  • 是否有正確的權限

Q: 怪物沒有生成？

A: 確認：

  • MythicMobs 怪物ID 是否正確
  • 生成位置是否有效
  • 世界是否存在
  • 檢查控制台錯誤信息

Q: 排行榜不顯示？

A: 檢查：

  • DecentHolograms 是否正常運作
  • PlaceholderAPI 是否安裝
  • 副本ID 是否正確
  • 是否有完成記錄

Q: 組隊系統無法使用？

A: 確認：

  • 組隊系統類型設置是否正確
  • 如使用 MMOCore，確保插件已安裝
  • 檢查相關權限設置
